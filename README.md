# Signal & Trace

Signal & Trace is a public-facing research and verification site built around investigative logic, source discipline, and current AI-assisted workflows.

The project is intentionally mobile-first and low-friction:

- static HTML pages
- Tailwind via CDN
- React via CDN only when a page needs interactivity
- direct GitHub editing from mobile
- no required build step

## Core positioning

The tools change. The method stays disciplined.

Signal & Trace is tool-agnostic. Gemini, NotebookLM, ChatGPT, Claude, Grok, Perplexity, browser tools, spreadsheets, PDFs, and manual search can all be part of the workflow. The brand is not locked to one model or platform.

## Site purpose

Signal & Trace helps people:

- verify claims
- check online entities
- structure public research
- build source-grounded case files
- separate facts, leads, assumptions, and unknowns
- turn scattered findings into usable outputs

## Main pages

- `index.html` — homepage and core positioning
- `tools.html` — interactive public research tools
- `blog.html` — research archive and guide entry point
- `case-reviews.html` — structured case review archive
- `ai-workflows.html` — current recommended AI research workflows
- `about.html` — detailed background and training
- `shane-lockhart.html` — personal SEO/profile page
- `contact.html` — contact and inquiry shell
- `build-reference.html` — internal mobile build reference

## Standard method

1. Define the target
2. Generate search paths
3. Capture useful returns
4. Preserve source context
5. Separate facts, leads, and assumptions
6. Map entities and timelines
7. Pressure-test contradictions
8. Build usable outputs
9. Update the workflow when better tools appear

## Important boundary

Signal & Trace is for lawful public-record and open-source research only. It is not a law enforcement portal, legal service, emergency service, or private investigation agency.
